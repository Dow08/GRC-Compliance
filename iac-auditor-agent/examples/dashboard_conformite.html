db_playbook.yml Database Security Hardening NON CONFORME HIGH Cliquer pour ouvrir

Constats techniques de l'auditeur

The configuration of PostgreSQL server exposes several security vulnerabilities, including: - **Plaintext password for superuser:** The database administrator's password is stored in plain text. This poses a significant risk as it can be easily compromised. - **Open listener address:** The database service listens on all network interfaces (0.0.0.0), making it vulnerable to unauthorized access. - **Disable TLS encryption for legacy clients:** Disabling TLS encryption for legacy clients is not recommended and increases the attack surface. This should only be done if there are specific reasons and proper mitigation strategies are in place. - **Trust access rule for subnet:** The trust access rule allows all users to connect without authentication, increasing the risk of unauthorized access.

Action de Remédiation Recommandée

Edit the configuration files to implement secure practices. For example: 
- **Password Management:** Use a strong password for the superuser and store it securely using a password manager or encryption tool.
- **Network Security:** Configure the database server to listen only on the internal network (e.g., 192.168.0.0/24) and restrict access to authorized users only.
- **TLS Encryption:** Enable TLS encryption for all connections to secure communication between clients and the database server.

Normes ISO 27001
ISO 27001:2022 A.8.24 (Use of Cryptography), ISO 27001:2022 A.8.15 (Logging), ISO 27001:2022 A.8.2 (User Access Rights)

Directive NIS2
NIS2 Article 21 (Data Security & Cryptography / Access Controls)

CVEs / CWEs
CWE-256, CWE-319, CWE-284

# FICHIPER PLAYBOOK DE BASE DE DONNÉES - CONFIGURATION VULNÉRABLE (POUR AUDIT DE SÉCURITÉ)
- name: Deploy PostgreSQL Database Service
  hosts: dbservers
  become: yes
  tasks:
    - name: Install PostgreSQL server
      ansible.builtin.package:
        name: postgresql-15
        state: present

    # INSECURITÉ : Mot de passe administrateur en clair dans le fichier de config
    - name: Initialize database superuser credentials
      postgresql_user:
        name: postgres
        password: "SuperSecretPassword123!"
        role_attr_flags: "SUPERUSER"

    # INSECURITÉ : Le service écoute sur toutes les interfaces réseau (0.0.0.0) au lieu de localhost/réseau interne
    - name: Configure database listener address
      ansible.builtin.lineinfile:
        path: /etc/postgresql/15/main/postgresql.conf
        line: "listen_addresses = '*'"
        state: present

    # INSECURITÉ : Désactivation explicite du chiffrement TLS (SSL) pour simplifier les connexions
    - name: Disable TLS encryption for legacy clients
      ansible.builtin.lineinfile:
        path: /etc/postgresql/15/main/postgresql.conf
        line: "ssl = off"
        state: present

    # INSECURITÉ : Règle d'accès réseau ouverte à la terre entière sans mot de passe (trust)
    - name: Configure trust access rule for subnet
      ansible.builtin.lineinfile:
        path: /etc/postgresql/15/main/pg_hba.conf
        line: "host all all 0.0.0.0/0 trust"
        state: present

Constats techniques de l'auditeur

The configuration disables TLS encryption, which leaves the database vulnerable to man-in-the-middle attacks. The use of MD5 for passwords is insecure and should be replaced with a strong cryptographic hashing algorithm like bcrypt or scrypt.

Action de Remédiation Recommandée

Constats techniques de l'auditeur

Missing NetworkPolicy to isolate the pod from other namespaces, leading to potential exposure and increased attack surface. Lack of resource limits (CPU/Memory) for containers, increasing risk of denial-of-service (DoS). - name: Deploy Secure SSH Service
  hosts: all
  become: yes
  tasks:
    - name: Ensure SSH server package is installed
      ansible.builtin.package:
        name: openssh-server
        state: present

    # CONFORMITÉ : Désactivation stricte de la connexion Root
    - name: Disallow root SSH login
      ansible.builtin.lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^#?PermitRootLogin'
        line: "PermitRootLogin no"
        state: present

    # CONFORMITÉ : Désactivation stricte de l'authentification par mot de passe (clés SSH obligatoires)
    - name: Disallow password authentication
      ansible.builtin.lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^#?PasswordAuthentication'
