Rapport d'Audit Technique

Écarts techniques identifiés (Gaps)
The configuration of PostgreSQL server exposes several security vulnerabilities, including: - **Plaintext password for superuser:** The database administrator's password is stored in plain text. This poses a significant risk as it can be easily compromised. - **Open listener address:** The database service listens on all network interfaces (0.0.0.0), making it vulnerable to unauthorized access. - **Disable TLS encryption for legacy clients:** Disabling TLS encryption for legacy clients is not recommended and increases the attack surface. This should only be done if there are specific reasons and proper mitigation strategies are in place. - **Trust access rule for subnet:** The trust access rule allows all users to connect without authentication, increasing the risk of unauthorized access.
Plan de Remédiation Technique Préconisé
Edit the configuration files to implement secure practices. For example: 
- **Password Management:** Use a strong password for the superuser and store it securely using a password manager or encryption tool.
- **Network Security:** Configure the database server to listen only on the internal network (e.g., 192.168.0.0/24) and restrict access to authorized users only.
- **TLS Encryption:** Enable TLS encryption for all connections to secure communication between clients and the database server.
